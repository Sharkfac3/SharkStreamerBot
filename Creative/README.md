# Creative

`Creative/` is for non-runtime creative scaffolding and content systems.

## Belongs here
- art-generation agents and references
- world-building and story scaffolding
- marketing scaffolding
- experiments, projects, references, and approved assets

## Does not belong here
- Streamer.bot runtime action scripts
- local tooling scripts and validators
- repo-wide architecture or workflow docs

## Key subfolders
- `Art/` — art-generation scaffolding, references, projects, assets
- `WorldBuilding/` — lore, settings, franchises, story systems, experiments
- `Marketing/` — future campaign and promo scaffolding

## Current structure
- `Art/Agents/`, `Art/Projects/`, `Art/Assets/`, `Art/References/`
- `WorldBuilding/Agents/`, `WorldBuilding/Franchises/`, `WorldBuilding/Storylines/`, `WorldBuilding/Settings/`, `WorldBuilding/Experiments/`
- `Marketing/Agents/`, `Marketing/Campaigns/`, `Marketing/Copy/`, `Marketing/Social/`

## Common routing examples
- Update an art prompt system → `Creative/Art/`
- Work on lore or franchise scaffolding → `Creative/WorldBuilding/`
- Add campaign or social copy systems → `Creative/Marketing/`
