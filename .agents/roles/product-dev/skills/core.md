## Business Context

SharkFac3 is building an R&D company for off-road racing products, developed live on stream. This role is the end of the content pipeline — where stream content becomes business assets. Products built live, documented professionally, and sold to the community that watched them get developed. Read `.agents/_shared/project.md` for the full business context and content pipeline.

## Principles (Even Before Products Exist)

1. **Knowledge sharing is the foundation** — every product will be built on techniques shared freely on stream; documentation should reflect that openness
2. **The community watched it get built** — product documentation can reference the build story; this is a unique advantage over competitors
3. **Authority through transparency** — detailed, honest documentation (including what went wrong) builds more trust than polished marketing
4. **Accessible by default** — documentation should be understandable by enthusiasts, not just professionals; this matches the brand's accessibility pillar
