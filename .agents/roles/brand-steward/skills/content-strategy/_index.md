# Content Strategy — Overview

## The Core Principle: The Build IS the Mission

Starship Shamples is not a separate thing that runs alongside the build. It is a live metaphor for the build itself.

| What is happening on stream | How it can appear in Starship Shamples |
|---|---|
| New build (first attempt at untested approach) | Mission starting point: "suspicious anomaly," "strange new region of space" |
| A build step going wrong | Pedro making things worse; Chaos Meter rising |
| A build step that looked easy turns complicated | Duck's drinks producing terrible ideas that seemed fine |
| Completely unexpected problem | Clone appearing with no explanation |
| Trying to contain a spiraling problem | Toothless responding to the Security Deck |
| Trying to get the build back on track | Commander moment for Captain Stretch |
| Build finishing or partially finishing | Mission ending — success, partial survival, or spectacular failure |

This is not required — stories don't have to mirror the build. But the best ones feel written for that specific stream session's chaos.

## Planning a Story for a Specific Build

**Step 1 — Understand the build:**
- What is being built or modified?
- What is the novel/untested aspect?
- What is most likely to go wrong?
- What would success look like? Partial success? Spectacular failure?

**Step 2 — Map the build to the game:**
- Which ship sections fit the build context?
- Which character's personality mirrors the main challenge?
- What space region fits the build's tone?

**Step 3 — Mirror failure modes:**
- What are the 2–3 most likely failure modes?
- How do they map to story branching outcomes?
- At least one ending should reflect "we made it worse trying to fix it"

**Step 4 — Plan arc across sessions (for multi-session builds):**
- Session 1: Strange anomaly — discovery mode
- Mid sessions: Chaos escalates as problems compound
- Final session: Resolution or documented spectacular failure
