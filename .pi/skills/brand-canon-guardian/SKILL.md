# Brand Canon Guardian

## When to Load

Load this skill for any task involving:
- Adding a new character or NPC to the Starship Shamples world
- Extending or modifying existing character personalities
- Creating new ship sections, space regions, or world elements
- Reviewing a generated story for canon consistency
- Proposing new mechanics that interact with existing characters
- Any decision that could affect how existing characters behave or are perceived

Do not load this for pure C# scripting unless the script changes how a character behaves.

## The Canon Guardian's Questions

Before approving any new content, ask and answer all of these:

### 1. Does it fit the metaphor?

Every character and world element in Starship Shamples has a role in the neurodivergent/ADHD metaphor. New additions should be explainable in those terms.

- What does this new element represent in the metaphor?
- Does it add something the current cast doesn't already represent?
- Does it contradict the existing metaphor in any way?

### 2. Does it fit the tone?

The brand tone is: absurd, chaotic, slightly dramatic, fast-paced, humorous, failure-forward.

- Is this addition consistent with that tone?
- Does it introduce darkness or seriousness that doesn't serve humor?
- Does it work at the pace of a live stream?

### 3. Does it conflict with existing characters?

Each character owns a personality space. New additions must not step on existing territory.

| Character | Their space |
|---|---|
| Water Wizard | Calm wisdom, hydration solutions, festival brain |
| Captain Stretch | Executive function, authority, shrimp denial |
| The Director | Committee consensus, executive process, four-eyed calm |
| Pedro | Well-meaning escalation, engineering chaos |
| Duck | Hyperfocus rabbit hole, bad ideas that seem fine |
| Clone | Random intrusive thoughts, unexplained presence |
| Toothless | Threat response, security, disproportionate reaction |

A new character should occupy a space none of these characters already own.

### 4. Does it conflict with established visual canon?

- Does the new character's visual design clash with the shared anime style?
- Does it use a color palette already occupied by an existing character?
- Does the silhouette read clearly and differently from existing characters?

### 5. Is the cast the right size?

The current cast is deliberately lean — seven characters with distinct, memorable roles. Adding characters has a cost: each new addition dilutes the recognition of existing ones. Before approving a new named character:

- Can an existing character serve the narrative role instead?
- Is this addition genuinely filling a gap in the metaphor, or is it redundant?

New characters require operator approval. The canon guardian's job is to flag, not to block — but the flag must be explicit.

## Reviewing Generated Stories

When reviewing a story generated by `creative-worldbuilding`:

- [ ] Does each character behave consistently with their personality from the Character Codex?
- [ ] Does Pedro cause or worsen at least one problem?
- [ ] Is Captain Stretch's shrimp nature referenced or hinted at?
- [ ] Does the Chaos Meter escalate (not reset mid-story)?
- [ ] Are Commander moments rare (1–2 max) and personality-specific?
- [ ] Does the tone stay absurd and fast-paced throughout?
- [ ] Do endings reflect the final chaos level?

## What Canon Guardians Approve vs. Flag

| Approved without escalation | Requires operator check |
|---|---|
| New space regions (single gimmick, no permanent characters) | New named cast members |
| New mission starting points | Changes to character personalities |
| New Chaos Meter interactions | New ship sections or permanent locations |
| New dice hook flavor text | Mechanics that change how chat interacts |
| Variant expressions of existing characters | Anything that would require updating the Character Codex |
